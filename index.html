This version is built to cut repetition. Features live in one backlog, each step inherits the last one, and the main outputs are organized as compact cards instead of long prompt blocks.

Cleaner output Each stage stores a structured record so the team can scan decisions fast instead of reading raw prompt text first.
Local-first Everything is saved in your browser on this machine, so your backlog stays available after refresh.
Expandable The data model is ready for future steps or API wiring when you want to turn prompt builders into true runs.
